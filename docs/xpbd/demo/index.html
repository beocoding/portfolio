XPBD Rigid Body Physics

Built in Rust · Bevy ECS · WebAssembly

Scene 1

Door Hinge

A dynamic door is attached to a static wall via a hinge joint constrained to ±90°. Gravity pulls it down while angular damping bleeds energy, letting it swing naturally to rest. The hinge enforces both positional and rotational alignment each sub-step.

Scene 2

Servo Arm

A purple arm hangs from a fixed anchor with a hinge that has a soft target angle of 45°. The constraint acts like a spring servo — gravity pulls the arm down while the position-based correction drives it toward the target, reaching equilibrium between the two forces.

Scene 3

Hinged Chain

Four rigid segments hang in series from a fixed orange anchor, each connected by a hinge limited to ±60° of swing. Drag the bottom segment to feel how forces propagate up the chain. Each joint independently enforces position, axis alignment, and angle limits every frame.

How it works

XPBD Algorithm Extended Position-Based Dynamics separates position correction from velocity updates, giving stable behavior at practical timesteps without energy blow-up.
Sub-stepped solver Each fixed frame is divided into 20 sub-steps. Constraints are solved iteratively within each sub-step, with compliance scaling by dt² to stay timestep-independent.
Bevy ECS pipeline Bodies are snapshotted from ECS into a plain Rust solver struct each frame, keeping the hot physics loop free of ECS overhead, then written back to transforms.
Inertia tensors Angular corrections use per-shape generalized inverse mass, accounting for rotational inertia in local space. Torques are applied as quaternion deltas, not Euler angles.
Anchor system Constraints attach at arbitrary local offsets via child anchor entities. World-space positions are computed each sub-step from the current body transform.
WebAssembly Compiled from Rust to wasm32 via Trunk. All solver internals use f32 for WASM performance. Rendered with Bevy's WebGL2 backend.

Try it: click and drag any of the colored bodies in the scene above. A soft grab constraint is created at the pick point and follows your cursor, letting you feel the weight and constraint behavior directly.
